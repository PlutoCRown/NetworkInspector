# 01 产品概览

## 1.1 产品定位

**NetworkInspector** 是一款面向埋点/网络请求调试的 **Chrome 扩展**。用户在目标网站上启用「规则组」后，扩展会：

1. 判断当前页面是否命中规则组；
2. 拦截并匹配配置的网络请求；
3. 从请求中按规则提取结构化字段；
4. 在**侧边栏**以卡片形式展示捕获结果。

产品同时服务两类用户：

| 角色 | 典型行为 |
|------|----------|
| **技术人员** | 编写/导出规则组 JSON、自定义渲染模板、配置高级解码 |
| **普通用户** | 从首页 Import 配置、一键启用、在侧边栏查看捕获结果 |

## 1.2 核心用户故事

### 普通用户

- 安装扩展后，看到引导：文档站、Playground 教程、Create、Import。
- 收到同事发来的规则组 JSON，Import 后立即在指定网站开始捕获。
- 通过工具栏快速开关当前规则组、打开侧边栏查看列表。

### 技术人员

- 在独立窗口（或设置页）中创建/编辑规则组：站点正则、捕获 URL、按接口配置提取与 Renderer。
- 为复杂响应（Protobuf、加密体）编写高级解码脚本。
- 编写 HTML 渲染模板，通过 `x-*` 指令控制标题、展开区、Popover 等行为。
- 导出 JSON + 可选模板文件，分发给团队。

## 1.3 规则组：四段流水线

每条规则组在逻辑上分为 **四个串联阶段**（详见 [03-rule-group-model.md](./03-rule-group-model.md)）：

```
站点匹配 (sites)
    → 请求捕获 (capture)
        → 结构化提取 (rules[].fields)
            → 侧边栏渲染 (React renderer)
```

任一阶段未命中则该请求**不进入**下一阶段；不得出现「未匹配站点却展示捕获卡片」等情况。

## 1.4 非目标（当前版本）

- 不作为通用 API 调试工具替代 DevTools Network 面板的全量功能。
- 不承诺云端同步、多人协作（可后续迭代）。
- 不提供对响应体的修改或重放（只读观察）。

## 1.5 成功标准（产品级）

- 导入示例 [`rule-group.schema.example.json`](../../rule-group.schema.example.json) 后，在匹配的 `acme.io` 页面上能捕获 `/v1/events` 与 `/v1/beacon`，侧边栏出现对应卡片。
- 普通用户无需写代码即可完成 Import → 启用 → 查看。
- 技术人员可通过高级模式覆盖解码与渲染，且与同一 JSON 模型兼容。
